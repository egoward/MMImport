Initial Configuration

First ensure the export target is set to "OGR".

OGR supports multiple backends.  We export the data as a number of layers within a single datasource.

ESRI Shape Files (.SHP)

We recommend you select the following options:

DriverName ESRI Shapefile
DataSourceName C:\temp\MasterMapOut
LayerName MM_{0}

By default, successive files will be overwritten.

To merge a large number of gz files into a single shape file, set "Append To Existing Files" to "True".

To import each input file into a collection of separate output file, set the LayerName to something like "MM_{0}_{1}".  {0} will be the layer name, eg 'TopographicArea' and '1' will be the source filename eg 'InitialSupply_47135-SS7886'.  Files will then have names like 'MM_TopographicArea_InitialSupply_47135-SS7886-5i1.shp'.

To automatically create directories for each input file, set "OutputEachFileToItsOwnDirectory" to "True".

Note that shape files use 32 bit integer file offsets internally - if you try to create a file more than 2Gb in size, problems will occur.

MapInfo TAB Files (.TAB)

We recommend you select the following options:

DriverName MapInfo File
DataSourceName C:\temp\MasterMapOut
LayerName MM_{0}

Additional options function as per the SHP file.

GML Files

DriverName GML
DataSourceName C:\temp\MasterMapOut\Map.GML
LayerName MM_{0}

GML files can contain multiple layers.

Use OutputEachFileToItsOwnDirectory to get each input tile individually.

Use AppendToExistingFiles to merge lots of input files together

KML Files

DriverName KML
DataSourceName C:\temp\MasterMapOut\Map.KML
LayerName MM_{0}

Options as per KML.  Attributes are output in the description.  No formatting information is generated.

 

Importing data using WGS84 / Latitude and Logitude

You can convert data from OS grid references (known as OSGB1936 or EPSG:27700) to Latitude and Logitude systems used by GPS (known as WGS84 or EPSG:4326). To do this, enable the "ConvertToWGS84" option to true and specify the SRID of 4326.

The conversion is different from that available in PostGIS and PostGIS does not include the OSTN02 dataset used by Ordnance Survey. If you're mainly working with WGS84/GPScoordinates, I recommend you use this option. If you're mainly working with data surveyed in the UK then it won't be appropriate. If you get it wrong, the errors in coordinate conversion will be of the order of 7 meters.
